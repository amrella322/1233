1. Тестова документація
Тестова документація — це набір документів, які описують процес тестування програмного забезпечення, сценарії перевірок та результати тестування.
Вона використовується для:
контролю якості продукту
планування тестування
фіксації знайдених дефектів
повторного тестування

2. Основні елементи тестової документації

2.1 Test Plan (План тестування)
Test Plan — документ, який описує загальну стратегію тестування.
Містить:
мету тестування
об’єкт тестування
типи тестування
тестове середовище
критерії початку та завершення тестування

2.2 Test Case (Тест-кейс)
Test Case — це детальний опис одного сценарію тестування.
Містить:
передумови
кроки тестування
очікуваний результат
фактичний результат
статус тесту (Pass / Fail)

2.3 Check List (Чек-лист)
Check List — це короткий список перевірок без детального опису кроків.
Використовується для:
швидкого тестування
регресійного тестування
перевірки простого функціоналу

2.4 Bug Report (Звіт про дефект)
Bug Report — документ, що описує знайдену помилку в програмі.
Містить:
короткий опис помилки
кроки для відтворення
очікуваний результат
фактичний результат
серйозність (Severity)
пріоритет (Priority)

3. Опис тестового додатку ListBoxer
ListBoxer — тестовий desktop-додаток для роботи зі списком.
Основний функціонал:
додавання елементів до списку
видалення елементів
очищення списку
робота з порожніми значеннями
робота з дублікатами

4. Test Plan для ListBoxer
Назва: Test Plan for ListBoxer
Мета: Перевірка коректності роботи функціоналу керування списком
Об’єкт тестування: Desktop-додаток ListBoxer
Типи тестування:
Функціональне тестування
UI тестування
Негативне тестування

Тестове середовище:
ОС: Windows 10 / 11
Тип додатку: Desktop
Критерії початку:
додаток встановлено та запущено
UI відображається коректно
Критерії завершення:
виконані всі тест-кейси
заведені всі дефекти

5. Check List
 Додаток запускається без помилок
 Можна додати елемент до списку
 Неможливо додати порожнє значення
 Обробка дублікатів працює коректно
 Можна видалити обраний елемент
 Неможливо видалити елемент без вибору
 Кнопка очищення списку працює
 Додаток не падає при швидких кліках
 UI коректно відображає довгі рядки

6. Test Cases
TC-01: Додавання елемента
Передумова: Додаток запущено
Кроки:
Ввести текст Test
Натиснути кнопку Add
Очікуваний результат:
Елемент Test з’являється у списку

TC-02: Додавання порожнього значення
Кроки:
Залишити поле введення порожнім
Натиснути Add
Очікуваний результат:
Елемент не додається, з’являється повідомлення про помилку

TC-03: Видалення без вибору елемента
Кроки:
Не обирати жоден елемент
Натиснути Remove
Очікуваний результат:
Додаток не завершується аварійно, з’являється попередження

7. Bug Reports (знайдені дефекти)
Bug-01: Додається порожній елемент
Опис: Порожній рядок додається до списку
Кроки відтворення:
Залишити поле введення порожнім
Натиснути Add
Очікуваний результат:
Порожній елемент не додається
Фактичний результат:
У список додається порожній рядок
Severity: Medium
Priority: High

Bug-02: Крах додатку при видаленні без вибору
Кроки:
Не вибирати елемент
Натиснути Remove
Фактичний результат:
Додаток аварійно завершується
Severity: Critical
Priority: High

Bug-03: Дублікати додаються без обмежень
Очікуваний результат:
Повідомлення або заборона додавання дубліката
Фактичний результат:
Однакові значення додаються без попередження
Severity: Low
Priority: Medium

Bug-04: Некоректне відображення довгих рядків
Кроки:
Ввести рядок понад 500 символів
Натиснути Add
Фактичний результат:
Текст виходить за межі елементу списку
Severity: Minor
